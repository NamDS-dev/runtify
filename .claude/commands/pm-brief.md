**오늘 진행 상황 + 내일 권장 작업**을 한국어로 요약해 Notion DB에 새 row로 기록합니다.

## 사용 시점
- 밤 작업 세션 종료 직전 (`cmd+ctrl+q`로 잠자기 들어가기 전)
- 또는 아침 출근 전 미리 확인하고 싶을 때 수동 실행

## 실행 절차

### Step 1: 상태 수집 (로컬)

다음 명령/파일을 읽어 데이터 취합:

1. `git log --oneline --since="1 day ago"` — 최근 24시간 커밋
2. `git status --porcelain` — 미커밋 변경 (개수만 파악)
3. `docs/STATUS.md` — 현재 진행 중 작업
4. `docs/FEATURE_PLAN.md` — 미완 Phase/기능 우선순위
5. `docs/DEVICE_TEST_QUEUE.md` — 실기기 검증 `[ ]` 개수
6. `docs/handoffs/` 최신 파일 1~2개 — 최근 작업 상세

### Step 2: 요약 본문 작성 (한국어, Markdown)

아래 섹션 구조로 간결하게:

```markdown
## 🎯 오늘/어제 진행
- (수동 작업 + 자동화 PM 세션의 성과 각각 bullet)
- (주요 커밋 해시 + 한 줄 설명)

## 📋 내일 권장 작업 (우선순위)
### 1. (가장 임팩트 큰 것, 🔴/🟡 아이콘으로 심각도 표시)
### 2.
### 3.

## 🐞 열린 이슈
- (미해결 버그만. 실기기 검증 대기는 여기 말고 따로 카운트)

## 🚀 자동화 진행률
- [x] Level 1/2/3/4 현황
```

### Step 3: Notion DB에 row 생성

`mcp__claude_ai_Notion__notion-create-pages` 호출:

```
parent: { "data_source_id": "47671339-7344-4cb4-8d26-b1917790c884" }
properties:
  "제목": "YYYY-MM-DD (요일)"        예: "2026-04-21 (화)"
  "date:날짜:start": "YYYY-MM-DD"     오늘 날짜
  "상태": "대기"                      (사용자가 확인 후 "확인됨"으로 변경)
  "실기기 대기": <number>             DEVICE_TEST_QUEUE의 [ ] 체크박스 개수
  "어제 진행": "<한 문장 요약>"
  "오늘 권장": "<우선순위 3개 한 줄 요약>"
  "최근 커밋": "<해시> <제목> | <해시> <제목>"  최근 2~3개
content: Step 2의 Markdown 본문
```

### Step 4: 사용자에게 한 줄 보고

아래 형식으로만 출력 (장황한 설명 금지):

```
📋 일일 브리핑 Notion 업로드 완료
🔗 <생성된 페이지 URL>
오늘 커밋 N개 · 실기기 대기 M건 · 내일 권장 작업 K건
```

## 규칙

- **덮어쓰기 금지**: 같은 날짜 row가 있어도 새 row 추가 (각 세션의 변화 추적 목적)
- **데이터 없음**: git 커밋이 없고 변경도 없으면 "어제 진행" = "작업 없음"으로 기록
- **과장 금지**: 실제로 완료된 것만 진행으로 간주. 코드 수정만 하고 검증 안 됐으면 "적용됨 (검증 대기)"로 명시
- **한국어 유지**: 전부 한국어로 작성 (영문 기술 용어는 허용)
- **실기기 대기 카운트**: `grep -c '^- \[ \]' docs/DEVICE_TEST_QUEUE.md` 로 정확히 세기
