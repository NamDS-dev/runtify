## 🟡 1. 이메일 인증(Verification) 정책

> 관련 구현: [FEATURE_PLAN.md — 이메일 인증 플로우](FEATURE_PLAN.md)

### 배경
- 현재 이메일 회원가입 시 인증 메일을 발송하지 않음 → 미검증 계정도 모든 기능 사용 가능
- OAuth(Google/Apple)는 인증된 이메일이 제공되므로 별도 검증 불필요

### 결정 필요 사항

| 항목 | 선택지 | 결정 |
|------|--------|------|
| **미인증 상태에서 허용할 기능** | (A) 조회만 / (B) 조회+개인 러닝 / (C) 거의 모든 기능 | ⬜ TBD |
| **미인증 상태에서 차단할 기능** | 크루 가입 / 리워드 교환 / 랭킹 등록 / 기타 | ⬜ TBD |
| **인증 메일 재발송 쿨다운** | 60초 / 120초 / 5분 | ⬜ TBD |
| **인증 유예 기간** | 즉시 강제 / 7일 유예 / 무기한 유예 | ⬜ TBD |
| **OAuth 가입자의 `emailVerified`** | 항상 true (자동 검증 간주) | ⬜ TBD |

### 구현 전제
- `UserModel`에 `emailVerified: bool` 필드 추가
- 홈 상단 배너 UI (미인증 상태일 때만 표시)

## 🟡 2. 로그인 실패 레이트 리밋 정책

> 관련 구현: [FEATURE_PLAN.md — 레이트 리밋](FEATURE_PLAN.md)

### 배경
- 현재 Firebase 기본 rate limit(IP 기반)에만 의존 → 계정 단위 브루트포스 방어 없음
- 로컬만 구현 시 앱 재설치로 우회 가능

### 결정 필요 사항

| 항목 | 선택지 | 결정 |
|------|--------|------|
| **실패 허용 횟수** | 3회 / 5회 / 10회 | ⬜ TBD |
| **잠금 시간** | 60초 고정 / 점진 증가(60→120→300) | ⬜ TBD |
| **서버 측 강화 여부** | 로컬만 / Cloud Functions 추가 | ⬜ TBD |
| **Blaze 요금제 전환** | 지금 / 서버 강화 필요 시점 / MVP 이후 | ⬜ TBD |
| **성공 시 카운터 리셋** | 즉시 리셋 / 24시간 후 자동 리셋 | ⬜ TBD |

### 구현 전제
- 이메일별 키로 `SharedPreferences` 저장 (예: `login_fail_count_{email_hash}`)
- UI에 남은 대기 시간 카운트다운 표시

## 🟡 3. 세션 만료 및 토큰 갱신 정책

> 관련 구현: [FEATURE_PLAN.md — 세션 만료 처리](FEATURE_PLAN.md)

### 배경
- Firebase ID token은 1시간마다 자동 갱신 시도 → 명시적 갱신 실패 대응 없음
- 러닝 중 토큰 만료 시 UX 시나리오가 가장 까다로움 (데이터 유실 가능)

### 결정 필요 사항

| 항목 | 선택지 | 결정 |
|------|--------|------|
| **유휴 강제 재인증** | 적용 안 함 / 30분 / 1시간 / 24시간 | ⬜ TBD |
| **러닝 중 토큰 만료 UX** | (A) 로컬 보존 + 복귀 시 재시도 / (B) 백그라운드 silent refresh만 / (C) 러닝 중엔 절대 로그아웃 강제 안 함 | ⬜ TBD |
| **토큰 갱신 실패 처리** | 즉시 로그인 화면 리다이렉트 / 3회 재시도 후 리다이렉트 | ⬜ TBD |
| **갱신 실패 UI** | SnackBar 안내 / Dialog 강제 / 조용히 리다이렉트 | ⬜ TBD |

### 구현 전제
- `FirebaseAuth.instance.idTokenChanges()` 스트림 모니터링
- `lib/core/services/session_manager.dart` 신설

## 🔴 4. 계정 탈퇴 정책

> 관련 구현: [FEATURE_PLAN.md — 계정 탈퇴 플로우](FEATURE_PLAN.md)
> App Store 14+ 정책상 **필수** — 미구현 시 심사 거절 가능성

### 배경
- 현재 프로필 페이지에 로그아웃만 있음. 탈퇴 기능 전무
- 크루 리더 탈퇴, 랭킹 기여 데이터 등 연관 데이터가 많아 정책 결정이 가장 복잡

|------|--------|------|
| **연관 데이터 처리** | (A) 완전 삭제 / (B) 익명화 (닉네임 "탈퇴한 러너"로) / (C) 소프트 삭제 | ⬜ TBD |
| **러닝 기록 처리** | 완전 삭제 / 크루·랭킹 집계만 유지 | ⬜ TBD |
| **크루 리더 탈퇴 시** | 최고참에게 리더 위임 / 크루 해체 / 탈퇴 제한(위임 후 탈퇴) | ⬜ TBD |
| **유예 기간** | 즉시 삭제 / 7일 복구 가능 / 30일 복구 가능 | ⬜ TBD |
| **재인증 요구** | 비밀번호 재입력 / 소셜 재로그인 / 생략 | ⬜ TBD |
| **GDPR/CCPA 문구** | 포함 / 추후 추가 / 불필요 | ⬜ TBD |
| **배지·업적 처리** | 삭제 / 크루에게는 이력 남김 | ⬜ TBD |

### 구현 전제
- 프로필 페이지에 "계정 삭제" 버튼 + 확인 다이얼로그
- `FirebaseAuth.currentUser.delete()`는 최근 로그인 세션 필요 → 재인증 유도
