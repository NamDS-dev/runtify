## 🎯 핵심 가설 (검증 대상)

### 가설 1 — 지역 기반 랭킹 경쟁 호기심
사람들이 자신이 사는 지역 기준 러닝 랭킹에 호기심을 갖고 자발적으로 참여하는가?

**측정 지표** (Firebase Analytics):
- 랭킹 탭 진입율 (홈 → 랭킹 클릭, DAU 대비)
- 랭킹 화면 체류 시간 (3초 이하면 호기심 X)
- 재방문 시 랭킹 확인 빈도 (두 번 이상 본 사용자 %)
- 정성: 인앱 피드백 reaction

### 가설 2 — 게임화된 캐릭터 성장 경험
달리기를 통해 "내 캐릭터가 성장한다"는 게임 같은 경험을 느끼는가?

**측정 지표**:
- 레벨업 직후 다음 러닝까지 시간 (짧을수록 강력)
- D7/D30 retention
- 배지 획득 시 공유/스크린샷 행동
- 정성: "더 뛰고 싶다" reaction

## 📋 Tier 2 — 출시 후 데이터 보고 결정

데이터/사용자 피드백 본 후 진행 여부 결정.

- **Anti-cheat / 데이터 신뢰성**
  - GPS 스푸핑 방지 (지역 랭킹 공격 발견 시)
  - 비현실적 페이스/거리 검출 (예: 시속 100km 자동차)
  - 트레드밀 vs 야외 구분
  - 트리거: 부정 신고 발생 또는 비정상 데이터 패턴 검출

- **커뮤니티 운영 (신고/차단/피드백)**
  - 욕설/스팸 신고 기능
  - 사용자 차단 기능
  - 인앱 버그/피드백 채널
  - 모더레이션 도구
  - 트리거: 첫 신고 발생 또는 사용자 100명 도달

- **시즌제 / 포인트 리셋**
  - 월별/분기별 리셋 메커니즘
  - 명예 전당 (전 시즌 1등)
  - 트리거: 포인트 인플레이션으로 가치 하락 발견

- **푸시 알림 확장**
  - 일일 미션
  - 친구 활동 알림
  - 휴면 사용자 복귀 알림
  - 트리거: Tier 1 푸시 1건 검증 후

## 🗑 Tier 3 — 컷 (가설 검증 우선)

가설 검증 결과 후 진행. **출시 전엔 안 함**.

### 소셜 기능 확장
- 1:1 친구 / 팔로우 / 팔로잉
- 다른 사용자 프로필 보기
- 활동 피드 / kudos / 댓글
- 크루 소셜 (게시글/댓글) — 이미 출시 후로 분리됨

### 수익화 / 경제 시스템
- 프리미엄 구독
- 광고 (AdMob)
- IAP (인앱 구매)
- 크루 유료 기능
- 브랜드 협업 챌린지

### 분석 / 인사이트 (고급)
- VO2 max 추정
- 회복 시간 / 훈련 부하
- 코칭 / 다음 러닝 추천
- AI 기반 페이스 분석

### 콘텐츠 / 학습
- 러닝 초보자 가이드
- 부상 방지 팁
- 스트레칭 / 워밍업 안내

### 국제화
- 영어 지원
- 미터법/마일 단위 설정
- 해외 시장 진출 (Trader 등록 등)

### 세부 UX
- 다크모드 외 테마 옵션
- 햅틱 피드백 강화
- 홈 화면 위젯
- 스마트워치 컴플리케이션

### 데이터 무결성
- 백업 / 복원
- 다중 디바이스 동기화
- 데이터 export (CSV/GPX)

### R2-R17 출시 후 작업
- 목표 페이스 알림 / 운동 종류 선택 / 결과 공유 / 수동 거리 보정 / iOS Live Activity / 음악 컨트롤 / 러닝 비교 / Year in Review / 응급 연락처 / Apple HealthKit·Strava·Garmin (FEATURE_PLAN.md 참조)

## 🤔 다음 브레인스토밍 주제 (피로 풀린 후)

### A. 레벨 공식 + 칭호 + 레이더 차트
- 레벨 공식 디테일 (지수 곡선 파라미터)
- 칭호 6단계 명명 (Lv.1/5/10/20/30/50)
- 레이더 차트 3축 (속도/지구력/꾸준함) 계산 방법

### B. 푸시 알림 전략
- 트리거 (랭킹 변동 / 친구 기록 / 일일 미션)
- 메시지 톤 (친근/도전/정보)
- 빈도 (하루 N회 상한)
- 시간대 (러닝 추정 시간 근처)

### C. Analytics 이벤트 설계
- 핵심 이벤트 6개 명세 (이벤트명/속성/발화 시점)
- Firebase Analytics 대시보드 설계
- A/B 테스트 가능 여부

### D. 랭킹 화면 first-impression 강화
- 빈 상태 처리 ("당신이 첫 도전자")
- next-step 표시 ("X km 더 뛰면 N위")
- 지역 캐치프레이즈
- 신규 유저 5초 안에 흥미 갖게 하는 메시지

### E. 큰 갭들 (이전 분석)
- 차별화 1줄 정의
- 첫 사용자 경험 (FTUE) 5분 안에 가치 인지
- 데이터 신뢰성 / Anti-cheat
- 커뮤니티 운영 백본
- Retention 메커니즘
- 소셜 그래프 (1:1 친구)
- 수익화 / 경제 시스템

## 💡 1인 개발자 원칙

- **완벽 X, 검증 우선** — 가설 검증 못 하면 모든 기능 의미 없음
- **시간이 가장 비싼 자원** — 컷 결정이 추가 결정만큼 중요
- **데이터로 다음 결정** — 직감 X, 사용자 행동 데이터 O
- **출시 전 = Tier 1만**, 출시 후 데이터 보고 Tier 2/3 결정
